Что такое Workflow?

Это способ как группа людей работает над проектом Обычно есть Главная Ветка "Main", эта самая стабильная версия которую обычно выпускают в релиз
На ней никто не работает,все изменения происходят предварительно на ветке "Develop"
Если сотруднику сказали сделать новую фичу в проекте, то он находясь на "Develop" создаёт новую ветку "Feature"
Когда фичу проверили на баги, её сливают с веткой "Develop"
В определенный момент когда было сделано несколько таких фич, создаётся ветка "Release"
Если в Релизной ветке всё проверено на наличие багов, её сливают с главной веткой "Main"
Но когда случаются казусы, когда всё жё баг не нашли и он проявился в главной ветке то, с неё создается ветка "hotfix"
Там фиксят баг и сливаются одновременно с "Develop" и "Main"

Git Rebase, Git Fetch

Git Rebase — это мощный инструмент в Git, который позволяет изменять историю коммитов путем перемещения вашей текущей ветки на другой коммит.
В отличие от операции слияния (merge) Rebase переписывает историю коммитов. Git Merge: создает новый коммит, который объединяет изменения из целевой ветки и исходной ветки. Обычно используется для интеграции финальных изменений или крупных новых функций.
Git Rebase: переносит ваши коммиты на конец другой ветки. Это создает иллюзию того, что изменения были применены на последний коммит в целевой ветке. Результат — линейная история коммитов без дополнительных коммитов слияния.
